# PRODUCT_SPEC — 每日零决策卡 (Daily Zero-Decision Card)

> 本文件定义**产品是什么、不是什么**。技术实现细节见 `TECH_SPEC.md`；交付标准见 `DELIVERY_STANDARD.md`。

---

## 1. 一句话定义

**每日零决策卡 = 一张最小的当日行动卡 + 一片会慢慢长出云的天空。**

- 它**不**是 Todo、**不**是番茄钟、**不**是习惯打卡器、**不**是数据看板、**不**是社交 App。
- 它的全部价值：**把"想坚持的事"压成今天能完成的一小步，并让你肉眼看见自己正在慢慢做下去。**

---

## 2. 核心机制（用户视角）

### 2.1 今日卡 (Today Card)

- 用户输入一段模糊的愿望（"想读书"、"想跑步"）。
- 应用把它**压成**一个最小行动：具体、很小、今天可完成。
- 一张卡只承载**一个**行动；做完即止。
- 完成后，云端会有一朵对应的云浮起来，永久留在用户的天空。

### 2.2 天空花园 (Sky Garden)

- 用户的天空以"云"的形式可视化完成历史。
- 每朵云 = 一次完成的行动。云不会消失，不会被惩罚。
- 提供 4 种云主题（阅读云 / 散步云 / 编码云 / 日常云），柔和油画布 + 软圆 3D 风格。
- 记忆云（5→7 朵）作为连续天数的视觉锚点。

### 2.3 回顾 (Stats)

- **自然语言回顾**，不堆图表。
- 看见"自己慢慢在坚持"，而不是"你今天又没做"。

### 2.4 安心卡 (Peace Cards, MVP 子系统)

- 用户库存最多 2 张安心卡。
- 用于保护中断日：被保护的日子**不会伪造 log**。
- 这是"允许不完美"的物化表达，不奖励也不惩罚。

### 2.5 天空宠物 (Sky Pet, cloud_cat MVP)

- 副线情感系统，让天空不只是数据图。
- 亲密度**只增不减**；不催、不责怪、不惩罚。
- 默认角色：小云。

### 2.6 番茄钟 (Pomodoro, 可选小工具)

- 降级为"需要时再用"的小工具。
- 不出现在主流程，不打断今日卡。

## 3. 显式排除（Anti-Scope）

### 3.0 功能准入规则（Feature Admission Rule）

> **任何新功能在被纳入需求池前，必须回答一个问题：它是否降低了用户的"决策成本"？**
>
> - 如果答案是**"是"**（让今天的一小步更小、让完成更轻、让坚持更静），可以进入评估。
> - 如果答案是**"否"**（它带来的是"更多选项 / 更多激励 / 更多可视化"），直接拒绝。
> - 如果答案是**"不确定"**，按"拒绝"处理 —— 增加一个无法证明降低决策成本的功能，默认为产品负债。

这条规则是 §3 Anti-Scope 表格之上的**判定原则**。表格列出的是"已知禁止"，本规则用于回答"未列出的新想法怎么办"。

### 3.1 显式边界（Hard Boundaries）

以下是**任何版本都不允许跨越的边界**，写在这里是为了让"加一个吧"在提出来的那一刻就能被直接否决：

- **NO accounts** — 没有用户账号、没有身份系统。
- **NO login / sign-up** — 没有登录、没有注册、没有第三方鉴权（OAuth / 短信 / 邮箱）。
- **NO cloud sync** — 没有云端同步、没有云备份、没有跨设备同步。手动导入导出是上限。
- **NO social** — 没有好友、没有关注、没有私信、没有协作、没有分享卡片。
- **NO rankings / leaderboards** — 没有排行榜、没有对别人的可见性、没有"我比 X% 的人更好"。
- **NO XP / points / badges** — 没有积分、没有经验值、没有徽章、没有解锁动画。
- **NO red warnings** — 没有红色警告、没有未完成红点、没有"今天还没做"的提示徽章。

> 任何一项若被提议加入且未被明确否决，**视为产品事故**，应回滚并修订本文件。

### 3.2 已知禁止清单（Anti-Scope）

以下功能在当前产品定位下永不做，写在这里是为了防止后续被"看起来很合理"的需求侵蚀：

| 不做 | 原因 |
| --- | --- |
| 用户账号 / 登录 / 注册 | 单设备单用户，不需要身份 |
| 云端同步 / 云备份 | 数据敏感、隐私优先；本地 `localStorage` 已足够 |
| 好友 / 关注 / 排行榜 / 协作 | 这是一个安静的自我陪伴 App，社交化会立刻污染氛围 |
| 推送通知 / 闹钟 / 催办 | 一切催办都违反"零决策"的核心承诺 |
| 积分 / 成就弹窗 / 连续打卡红点 | 任何"奖励机制"会引入焦虑，是反 PUA 铁律的红线 |
| 数据看板 / 折线图 / KPI | 用户要的是"自然语言看见自己"，不是仪表盘 |
| 心情打分 / 打卡天数排行 | 量化自我 = 把内在动机外包给数字 |
| 主题商城 / 会员 / 付费 | 个人项目，不引入商业化压力 |
| Web Clipper / 浏览器扩展 | 与"一屏完成"冲突 |
| 同步到日历 / Notion / Webhook | 跨界集成会引入外部依赖与失败模式 |

## 4. 声音与语气 (Tone)

### 4.1 反 PUA 铁律（产品级）

产品**不**说以下任何话：

- "你今天还没开始呢"
- "再坚持一下就好了"
- "你已经连续 N 天了，不要断哦"
- "完成度只有 X%"
- "别忘了你的目标"
- 任何形式的红点、徽章、奖励弹窗

### 4.2 应该有的语气

- 平静、温和、像坐在窗边的朋友。
- 完成后给一朵云、一次呼吸的留白、一句"今天这样就够了"。
- 没完成时，应用不主动评价；用户可以自己点开回顾。

### 4.3 视觉基调

- 暖奶油画布底色（`#FFF7F0`），主题色柔绿（`#4AB574`）。
- 软圆 3D (claymorphism) 卡片，云主题壳。
- 减少动效尊重 `prefers-reduced-motion`。

## 5. 用户故事（边界用例）

| # | 用户场景 | 应用反应 |
| --- | --- | --- |
| 1 | 第一次打开 | 一屏式 Onboarding，2 步内可进入今日卡 |
| 2 | 输入"想读《置身事内》" | 解析为最小行动卡（页数+具体页码） |
| 3 | 完成今日卡 | 天空长出一朵阅读云，亲密度 +1，今日页不再出现"再养一朵"按钮 |
| 4 | 想换更轻的卡 | 提供 4 态 MAX 守卫，最多换一次 |
| 5 | 断了一天 | 静默；不弹窗、不扣分、不红点 |
| 6 | 用了 1 张安心卡 | 被保护日不写入 log；库存 -1 |
| 7 | 想看历史 | 回到"我的天空"，看见云；进入"回顾"看到自然语言 |
| 8 | 换手机 / 浏览器 | 数据在新设备不可见（无同步）；用户可手动导入导出 |
| 9 | 重置 | 设置页有显式"重置"，二次确认 |
| 10 | 飞行模式 / 离线 | 全部功能可用（PWA 离线优先） |

## 6. 成功定义（非数字）

产品成功的标志：

1. 用户**每天**能完成一卡，且不觉得被打扰。
2. 三个月后，用户能说出"我慢慢在坚持读书"，而不是"我已经连续 90 天了"。
3. 没有任何时刻让用户产生"我今天又没做到"的挫败感。
4. 用户愿意把应用放在首屏，不删除。

> **反 KPI 列表**：DAU / 留存 / 推送点击率 / 完成率 — 这些指标如果被用作优化目标，会立刻把产品推向 Duolingo 式焦虑驱动。**禁止纳入产品决策。**

## 7. 边界声明（给未来的自己）

如果有人（包括未来的我）提出以下需求，请先读本节：

- "加个排行榜吧" → 拒绝。见 §3。
- "加个推送提醒吧" → 拒绝。见 §3。
- "加个红点提醒未完成吧" → 拒绝。见 §4.1。
- "加个成就系统吧" → 拒绝。见 §3。
- "加个账号同步吧" → 拒绝。手动导入导出是上限。
- "把数据搬到云上吧" → 拒绝。本地优先是隐私底线。

如果以上"加"在某个版本中出现且未被明确否决，**视为产品事故**，应回滚并修订本文件。
