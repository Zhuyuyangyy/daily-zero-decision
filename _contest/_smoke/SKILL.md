# 公文格式检查与材料助手

## 适用场景

当用户需要处理以下材料时使用本 Skill：

- 通知、请示、报告、函、通报等正式公文
- 会议纪要、工作简报、活动通知、学生组织通知、行政办公材料
- 需要检查标题、主送机关、正文层级、落款、日期、附件和正式语气的文本
- 需要快速生成通知、请示、函、会议纪要等规范草稿的文本需求
- 需要从会议材料中提取责任单位、截止时间、待办事项的文本
- 需要处理空输入、短文本、坏 JSON、控制字符等非理想输入但不能崩溃的场景

## 工作流程

1. 判断用户材料类型：公文审查、模板起草、会议纪要/督办提取或政策要点梳理。
2. 优先读取用户直接提供的文本；如用户给出文件路径，读取 UTF-8 文本或 Markdown 文件。
3. 当用户需要起草时，根据文种、主题、主送机关、落款单位、截止时间和要求生成可编辑模板。
4. 对正式公文执行结构检查：
   - 标题是否符合“关于……的文种”习惯。
   - 是否有主送机关、正文、附件、落款单位和落款日期。
   - 是否存在口语化表达、强制性表述缺少依据、事项不明确等问题。
5. 对会议纪要和简报提取督办事项：
   - 责任方
   - 任务描述
   - 截止时间
   - 原文依据
6. 输出模板草稿或结构化审查报告，包含评分、风险等级、问题清单、修改建议和督办事项。

## 可用脚本

本 Skill 附带一个纯 Python 标准库脚本：

```bash
python scripts/gov_doc_review.py --mode review --format markdown < examples/input-notice.md
python scripts/gov_doc_review.py --mode brief --format json < examples/input-meeting.md
python scripts/gov_doc_review.py --mode draft --format markdown < examples/input-draft-notice.json
python scripts/gov_doc_review.py --mode draft --format markdown < examples/input-draft-meeting.json
```

参数说明：

- `--mode review`：公文格式和合规审查。
- `--mode draft`：根据 JSON 或自然语言需求生成公文模板。
- `--mode brief`：会议纪要/简报督办提取。
- `--format markdown`：输出适合直接粘贴的审查报告。
- `--format json`：输出结构化结果，便于二次处理。

## 输出要求

默认输出 Markdown 报告，结构如下：

1. 文种识别、综合评分、风险等级。
2. 标题、主送机关、落款单位、落款日期等结构识别结果。
3. 问题清单：每项包含严重程度、证据和修改建议。
4. 督办事项：责任方、期限、事项。
5. 规范化修改建议：保持正式、清楚、可执行，不编造不存在的政策依据。

起草模式输出可直接修改的 Markdown 正文，默认保留 `XXXX年XX月XX日` 等占位符，避免编造未提供的信息。

## 示例

示例输入：

```text
关于加强会议材料报送工作的通知

各部门：

为进一步规范会议材料报送工作，请各部门于2026年6月20日前完成材料汇总，并报办公室备案。

附件：材料报送清单

办公室
2026年6月10日
```

示例输出见 `examples/output-notice.md`。更多通知、请示、函、会议纪要起草示例见 `examples/input-draft-*` 与 `examples/output-draft-*`。

## 已知限制

- 本 Skill 是参赛轻量版，重点覆盖常见中文公文和办公室材料，不替代人工法务、公文专员或保密审查。
- 当用户要求生成政策依据时，不要编造法规名称；只能基于用户提供的政策文本或明确的通用公文规范提出建议。
- 涉及涉密、个人隐私或未公开政务数据时，只给本地处理建议，不建议上传外部平台。
