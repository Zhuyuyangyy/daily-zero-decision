# official-document-assistant 作品介绍

## 核心定位

official-document-assistant 是一款面向政务、行政办公和学生组织正式材料场景的公文格式检查与材料起草助手。它不是泛泛聊天工具，而是围绕“正式材料怎么写、格式哪里错、会议事项谁来办”三个具体痛点，提供可直接落地的 Skill 能力。

## 解决的问题

很多基层办公材料并不复杂，但容易卡在细节上：标题不像公文、主送机关漏写、落款日期缺失、语气太口语、会议纪要写完后任务没人跟。official-document-assistant 把这些高频问题拆成结构化检查规则和模板生成流程，让用户先得到一个规范骨架，再进入人工修改和复核。

## 主要能力

1. 公文审查：自动识别通知、请示、报告、函、会议纪要等类型，输出结构识别、问题清单、风险等级和修改建议，覆盖标题、主送、落款、日期格式和附件说明等要素。
2. 模板起草：支持 JSON 和自然语言输入，生成通知、请示、函、会议纪要草稿，保留必要占位符，避免编造未知信息。
3. 督办提取：从会议纪要中提取责任方、截止时间和任务描述，把会议要求转成可跟踪事项。
4. 稳定兜底：面对空输入、短文本、坏 JSON、控制字符和字段缺失时不崩溃，输出可解释的风险提示或占位模板。
5. 安全提示：识别疑似手机号、身份证号、个人信息、涉密和内部资料表述，提供本地脱敏预览，并提醒脱敏、本地处理和人工复核。

## 应用价值

对政务场景，它能降低公文起草和初审成本，提升材料处理效率；对办公室、行政、人事、学生会和社团场景，它能把低频但刚需的正式写作变成可复用流程。作品既贴合政务公文智能处理赛题，也具备热度榜传播空间，可以用“体制内/行政/学生会公文材料助手”的方式触达更广泛用户。

## 技术特点

作品采用轻量化 Skill 包结构，核心脚本只依赖 Python 标准库，便于在本地或 AstronClaw 环境部署。处理链路包括输入清洗、文种识别、结构抽取、规则审查、风险评分、模板生成、督办提取、脱敏预览和 Markdown/JSON 输出。v0.2 已通过 15 个自动化测试，并提供通知、请示、函、会议纪要多文种示例。

## 一句话卖点

把通知、请示、函和会议纪要变成可检查、可起草、可跟进的办公材料流程。
