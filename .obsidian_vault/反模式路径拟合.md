# 反模式: 路径拟合

> provenance audit中的p-hacking

---

## 什么是路径拟合

1. 数字在标称源文件找不到
2. 换个文件试试 ✓
3. 换个量表试试 (5/5 → 100) ✓
4. "凑到了" → 报告MATCH

**这就是provenance-level p-hacking**

---

## 正确做法

### 规则先定（不能从结果反推）

```python
# 规则1: 源路径固定
self_source = "<project>/_Q2_v1.md"

# 规则2: MATCH = "X/100"字面匹配
pattern = r'\bN\b\s*/\s*100'

# 规则3: NO format conversion
6.33/10 ≠ 80/100
```

### 找不到就停止

```python
if not found:
    return UNTRACEABLE  # 不再尝试
```

## 实际案例

### Cross-Lingual-TCM-Translator self=60

- 在任何项目.md中0次出现
- 不能"可能来自5/5小量表换算"
- 正确分类: **UNTRACEABLE**

### BioSynth-DBTL indep=65

- jsonl写65
- 源文件有32.65
- 正确分类: **MISMATCH (不是FORMAT-MISMATCH)**
