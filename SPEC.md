# 每日零决策卡 - 规格文档

## 0. 定位（一切设计的根）

> **治愈不焦虑的打卡。你在养天空，天空不会 PUA 你。**

- 反 Duolingo：漏签不骂，不施压
- 天空是 streak 的隐喻：数字 → 一片会变丰盈的活物
- 每朵云由日期 hash 生成 = 每天独一无二 = 天然可晒

## 1. 项目概述

**项目名称**：养一片自己的天空 · 每日零决策卡
**项目类型**：单页应用 (SPA)，纯前端，无后端
**核心价值**：帮人坚持完成任务，不是记笔记

### 设计系统参考
- **impeccable** 设计标准：界面打磨、视觉层次、动效微交互
- **taste-skill** 审美规范：配色、排版、治愈风格
- **Duolingo / Kami / Friendly** 设计系统：圆胖、温暖、治愈

### 视觉风格
- **配色**：暖奶米背景 (#FAF7F2) + 薄荷绿 accent (#86efac) + 珊瑚粉 (#fca5a5)
- **字体**：Plus Jakarta Sans (标题) + Nunito (正文)
- **圆角**：按钮 12-16px、卡片 20px、pill 9999px
- **阴影**：极简柔和，hover 时微提升

## 2. 核心功能循环

### 2.1 今日页（默认视图）
- 显示云朵吉祥物（平静态）
- 如果今天还没建卡：
  - 输入框：让用户输入任务描述
  - 预设 Chip：看书 / 背单词 / 写代码 / 运动 / 下课看书（默认）
- 如果今天已建卡：显示任务卡片

### 2.2 生成零决策卡
输入一句话后，自动生成：
- **任务**：描述
- **对象**：哪本书/项目（从输入推断）
- **起始页**：自动记住上次看到第几页，填入下一段起始页
- **结束页**：自动算到第几页
- **地点**：默认"安静的地方"
- **时间**：默认"30分钟"
- **完成方式**：「完成」按钮

### 2.3 任务卡片视觉
- 大字标题
- 极简留白
- 醒目的「完成」按钮
- 「截图/打印此卡」入口

### 2.4 完成打卡
- 页码前移
- 连续天数 +1
- 云朵从平静变开心 + 庆祝微动画
- 连续天数数字弹出
- 刷新后数据持久

---

## 3. 坚持机制

### 3.1 断一天不清零
- 漏打卡只当天不 +1
- 保留一天宽限补救
- 界面绝不放红色/警告/责备语气

### 3.2 文案调性
- 温柔鼓励，不施压
- 第二人称，像朋友说话
- 例如："今天辛苦了" "慢慢来，不着急"

```typescript
interface Task {
```

## 5. UI 组件设计

### 5.1 云朵吉祥物 (SVG + CSS)
**三种情绪态**：
1. **平静态**：闭眼微笑，轻微呼吸动画
2. **开心态**：睁眼大笑，跳跃动画
3. **成长态**：变大一点，闪烁星星

**动画规格**：
- 呼吸：`scale(1) → scale(1.02)` 3s 循环
- 开心：上浮 + 缩放 500ms back-out
- 庆祝：旋转 + 星星飞出

### 5.2 预设 Chip
- `border-radius: 9999px`
- 背景：柔和粉彩（薄荷/樱花粉）
- hover：轻微放大 + 阴影

### 5.3 任务卡片
- 大圆角 (20px)
- 极简内容
- 主 CTA「完成」按钮：厚底阴影 + 按下动效

### 5.4 连续天数展示
- 当前连续 X 天（大数字）
- 最近 7-14 天小圆点日历
- 绿色实心 = 已完成，灰色空心 = 未完成

## 7. 动效规格

### 7.1 时长规则
- 瞬时反馈：50-100ms
- 状态确认：150ms
- UI 出现：200-300ms
- 跨屏转场：300-500ms

### 7.2 缓动曲线
```css
--ease-out-quart: cubic-bezier(0.25, 1, 0.5, 1);  /* 推荐默认 */
```

### 7.3 禁止使用
- bounce / elastic 动画（过时）
- linear / ease-in-out 默认
- 纯黑 (#000) / 纯白 (#fff)

## 8. 天空系统（v0 改造重点）

### 8.1 设计原则
- **砍掉一切数字 streak UI**——连续天数不再是炫数字，而是养出一片天空
- 天空配色 = 当前坚持长度的隐喻
- 每朵云 = 一个已完成日期的纪念物
- 漏签 = 多云的一天，不死亡、不重置

### 8.2 天色映射

| 当前连续天数 | Sky Mood | 颜色氛围 |
|---|---|---|
| 0-1 | dawn | 暖米微光 |
| 2-6 | morning | 浅蓝 + 暖底 |
| 7-13 | clear | 晴空蓝 |
| 14-29 | sunny | 暖阳光晕 |
| 30+ | golden | 夕阳霞 |

**漏签软化**：最近漏签不会强退到 dawn，最多回退一档（sunny→clear），保持温柔过渡。

### 8.3 云朵生成（云朵是每日纪念物）

每朵云 = 由日期字符串 hash 出的 6-8 个视觉属性：

```
size, cx, cy, rotation, rx, ry, bumpCount, bumpSize,
hue, saturation, lightness, opacity,
expression, drift, floatSpeed
```

- 同一日期 → 永远同一朵云
- 不同日期 → 不同云
- 8 种表情：calm / smile / sleep / wink / tiny-smile / peeking / peaceful / neutral

### 8.4 可见云朵数
- 最多展示最近 12 朵（防 clutter）
- 最新的云带柔光 pulse
- 漏签的云饱和度降一档（视觉上"飘远了一点"，但仍然在）

## 9. 温柔文案系统（不骂你的云）

### 9.1 硬规则
- ❌ 失败、断了、再坚持、自律、加油、不能断
- ✅ 慢慢来、没关系、你来了、云说辛苦了

### 9.2 场景文案

| 场景 | 文案示例 |
|---|---|
| 第一次打开 | "今天，养下第一朵云吧" |
| 完成 | "第一朵云，归位了" / "今天的这一朵，挺好看的" |
| 30 天+ | "这片天空，是你亲手养出来的" |
| 漏签回来 | "没关系，云会飘回来" / "天空还在这里等你" |
| 当天未完成 | "今天想养什么样的云？" / "不用想太多" |
| 底部 | "放下手机，去做吧" |

### 9.3 文案系统文件
`src/utils/copy.ts` — 集中管理所有用户可见文案，便于审查和迭代。
