# 安心卡功能开发计划

## 功能概述

**「安心卡」**：允许用户休息一天，但不否定之前的坚持

## 核心原则

1. **不伪造log** - 安心卡只保护streak，不写假完成记录
2. **不贩卖焦虑** - 产品调性是柔软、不焦虑、允许休息
3. **不收费** - MVP阶段免费赠送，不做付费UI

## 数据模型

```ts
interface PeaceState {
  cards: number;           // 安心卡数量，最多2张
  protectedDates: string[]; // 被保护的日子（不伪造log）
  lastRewardedDate: string | null;  // 上次奖励日期
}
```

## 安心卡规则

| 规则 | 说明 |
|------|------|
| 新用户默认 | 2张安心卡 |
| 奖励规则 | 连续7天回来，奖励1张 |
| 上限 | 最多2张 |
| 消耗 | 断签时自动消耗1张 |
| 效果 | streak不断，但不写假log |

## UI组件

| 组件 | 说明 |
|------|------|
| PeaceCard | 显示安心卡数量（绿色背景） |
| PeaceCardInfoModal | 说明弹窗（非购买） |

## 文案原则

禁止：
- ❌ "断签焦虑"
- ❌ "努力白费"
- ❌ "立即购买"
- ❌ "天空投资"
- ❌ 任何价格/订阅

应该：
- ✅ "忙的时候，它会帮你温柔保住连续记录"
- ✅ "漏一天也没关系，云不会骂你"
- ✅ "明天回来也没关系"
