# official-document-assistant v0.3 升级说明

## 升级目标

v0.3 不扩大宣传范围，而是深化一个核心承诺：从“检查公文有没有明显问题”，升级为“给出可直接修改的规范修订稿”。

## 主要变化

- 新增“建议修订稿”输出，审查报告可直接复制后人工复核修改。
- 增强公文规则：标题三要素、正文层级、附件编号、请示结语、请示一文一事、报告夹带请示事项、函的商洽语气、通知可执行性。
- 优化请示、函、报告起草模板，减少机械占位表达。
- 支持 `.docx` 输入，使用 Python 标准库提取 Word 正文。
- 支持 `--batch folder` 批量审查和 `--output report.md` 文件输出。
- README 和 SKILL.md 删除未实现的“政策要点梳理/政策问答”承诺，避免能力虚标。
- 根据复审反馈修复 4 个观感问题：请示模板不再出现“申请申请”，会议纪要修订稿不再套通知格式，标题规则允许“发文机关 + 关于 + 事由 + 文种”，报告规则补充“拟申请/申请追加/请求批准/请予支持”等申请性表述。

## 与 v0.2 的关系

v0.2 已发布，可作为稳定参赛占位版本保留。v0.3 是后续冲专家榜和作品迭代的增强版本，重点提升结果质量、落地感和工程完整性。

## 验证项

- Python 语法编译通过。
- Pytest 覆盖建议修订稿、报告/请示规则、标题三要素、附件编号、docx 输入、batch 输出、敏感信息脱敏和空输入兜底。
- 示例输入输出已按 v0.3 重新生成。
